# 💳 신용 평가 승인 예측 인공지능 모델링 프로젝트

본 프로젝트는 고객의 금융 데이터를 분석하여 대출 승인 여부를 예측하는 인공지능 모델을 개발하고, 다양한 알고리즘의 성능을 비교 분석한 결과 보고서입니다.

---

## 1. 🎤 인공지능 모델 라인업 (솔로 vs 아이돌 그룹)
기존 프로젝트에 구성되어 있던 기본 모델들 외에, **분석의 객관성과 성능 차이를 더욱 명확히 확인하기 위해 `의사결정나무(Decision Tree)`와 `CatBoost` 2가지 모델을 추가**하여 총 6가지 모델로 실험을 진행했습니다.

### 👤 단일 모델 (Solo Algorithm)
> **비유: "한 명의 전문가가 자신의 논리로 판단합니다."**
*   **로지스틱 회귀 (Logistic Regression):** 가장 기초적인 논리 모델로, 분석의 기준점(Base) 역할을 합니다.
*   **의사결정나무 (Decision Tree) [추가]:** **[직접 추가한 모델]** 스무고개 방식으로 판단하며, 단일 모델 중 결과 도출 과정을 설명하기 가장 용이하여 비교 기준으로 활용했습니다.

### 👯 앙상블 모델 (Ensemble Algorithm / Idol Group)
> **비유: "여러 전문가가 팀을 이루어 서로의 약점을 보완하며 판단합니다."**
*   **랜덤 포레스트 (Random Forest):** 수백 명의 나무 모델이 투표로 결정하는 전통적인 강자입니다.
*   **XGBoost:** 틀린 문제를 복습하며 실력을 키우는 똑똑한 1등 후보 팀입니다.
*   **LightGBM:** 대량 데이터를 매우 빠르고 효율적으로 처리하는 퍼포먼스 팀입니다.
*   **CatBoost [추가]:** **[직접 추가한 모델]** XGBoost의 강력한 대항마로 불리는 최신 실력파 팀입니다. 앙상블 모델 간의 정밀한 성능 비교를 위해 특별히 영입했습니다.

---

## 2. 📏 모델 성능 측정 기준 (Metric)
인공지능의 실력은 단순히 맞히는 것뿐만 아니라, **'얼마나 균형 있게 맞히는가'**가 중요합니다.

1.  **정확도 (Accuracy):** 100문제 중 몇 개나 맞혔나?
2.  **정밀도 (Precision):** 모델이 "승인"이라고 예측한 사람 중 **진짜 승인 대상자가 몇 명인가?** (은행의 리스크 관리 지표)
3.  **재현율 (Recall):** 진짜 승인 대상자 중 **모델이 놓치지 않고 찾아낸 비율은 얼마인가?** (고객 확보 지표)
4.  **F1-Score (균형 점수) ★핵심 지표**: 정밀도와 재현율의 균형을 보는 점수입니다. 대출 심사처럼 편향이 중요한 분야에서 가장 신뢰할 수 있는 기준입니다.
5.  **ROC-AUC (종합 점수):** 모델이 실력으로 정답과 오답을 얼마나 잘 구별해내는지 측정하는 종합 성능 지수입니다.

## 🔍 프로젝트 분석 결론

1.  **추가 모델의 의의:** 기존 샘플 모델 외에 **`의사결정나무`와 `CatBoost`를 추가**함으로써, 가장 기초적인 모델(Solo)부터 최신 성능의 모델(Ensemble)까지 폭넓은 스펙트럼의 비교가 가능해졌습니다.
2.  **앙상블 모델의 우수성 확인:** 실험 결과, 추가한 `CatBoost`를 포함한 모든 **앙상블 모델**들이 단일 모델 대비 전반적으로 높은 예측력을 보였습니다. 
3.  **최종 모델 검토:** 지표상으로는 **XGBoost (튜닝)** 모델이 본 데이터셋에서 가장 높은 균형 점수(F1)를 기록하였습니다. 따라서 해당 도메인에서 예측 시스템 구축 시 **우선적으로 고려해 볼 수 있는 유력한 대안**으로 판단됩니다.

## 🚀 향후 연구 및 테스트 과제 (Future Work)

본 프로젝트의 성능을 한 단계 더 끌어올리고 실제 비즈니스에 적용하기 위해 다음과 같은 추가 연구 예정입니다.

### 1. 임계값(Threshold) 최적화: "은행의 대출 승인 기준점 찾기"
*   **연구 목표:** 현재 0.5(50%)로 고정된 승인 기준점을 비즈니스 상황에 맞게 조정합니다.
*   **상세 내용:** 
    *   **공격적 영업:** 기준을 0.3으로 낮춰 더 많은 고객에게 대출을 해주고 이자 수익을 극대화하는 실험.
    *   **보수적 관리:** 기준을 0.7로 높여 미상환 위험(연체 등)을 최소화하는 실험.
*   **확인 지표:** 정밀도(Precision)와 재현율(Recall)의 변화를 관찰하여 가장 적절한 '손익 분기점'을 찾아냅니다.

### 2. 변수 중요도(Feature Importance) 분석: "인공지능의 판단 근거 파악"
*   **연구 목표:** 인공지능이 신용 점수를 매길 때 **가장 중요하게 생각하는 데이터**가 무엇인지 파악합니다.
*   **상세 내용:** 
    *   XGBoost나 CatBoost의 '변수 중요도' 기능을 사용하여 연봉, 재산, 예금 중 어떤 항목이 결정적인지 확인합니다.
    *   예상 밖의 중요 변수(예: 자동차 배기량 등)가 있다면 그 이유를 논리적으로 분석해봅니다.
*   **확인 지표:** 각 데이터 항목별 기여도 점수 및 시각화 그래프.

### 3. 설명 가능한 AI (XAI) 도입: "거절 사유 설명하기"
*   **연구 목표:** 단순히 "거절"이라고 통보하는 대신, **"왜 거절되었는지"** 고객에게 설명해줄 수 있는 시스템을 구상합니다.
*   **상세 내용:** 
    *   SHAP(Shapley Additive Explanations) 라이브러리를 활용하여 개별 고객의 결과에 영향을 준 항목들을 수치화합니다.
    *   "연봉은 충분하지만 재산 대비 신청 금액이 너무 커서 거절되었습니다"와 같은 구체적인 피드백 생성 연구.
*   **확인 지표:** 개별 예측 결과에 대한 기여도 분석 리포트.

### 4. 파생 변수(Feature Engineering) 생성: "새로운 금융 지표 만들기"
*   **연구 목표:** 기존 데이터를 조합하여 인공지능이 더 잘 이해할 수 있는 **새로운 핵심 지표**를 만듭니다.
*   **상세 내용:** 
    *   단순 연봉이나 신청금액 대신 `부채 비율 = 신청금액 / (재산 + 예금)`과 같은 비율 데이터를 생성하여 학습에 추가합니다.
    *   인공지능이 새로운 지표를 통해 더 정확한 판단을 내리는지 검증합니다.
*   **확인 지표:** 파생 변수 추가 전후의 F1-Score 및 ROC-AUC 개선 폭.
